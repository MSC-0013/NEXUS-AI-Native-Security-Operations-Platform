# NEXUS — Engineering Completion Audit & Execution Plan

## Context

NEXUS is an AI-native SOC platform (React 19 + TanStack / Fastify 5 + Drizzle / PostgreSQL RLS).
This document is a **completion audit**: it enumerates the concrete implementation work required for
the *documented* architecture (PROJECT-DOCUMENTATION.md, ARCHITECTURE-DEEP-DIVE.md) to be considered
100% built and production-ready. It is **not** a feature-gap wishlist and **not** future enhancements —
only work needed to finish what is already designed.

**Ground-truth method:** verified the actual code (33 registered backend modules in
`apps/api/src/modules`, 45+ frontend routes in `src/routes`, 49 Drizzle tables in
`packages/db/src/schema.ts`, 11 SQL migrations, `src/lib/api-hooks.ts` ≈98 hooks). The prior
`AUDIT-REPORT.md` (2026-06-08) is partially stale — Cases CRUD, AI endpoints, suppression rules,
policies, incident escalate/rca/postmortem, vuln patch, investigation notes now **exist**. This audit
supersedes it with the current state.

**Top systemic findings (apply platform-wide):**
1. **No automated tests** anywhere (0 `.test`/`.spec` files). **No CI** (`.github/workflows` absent).
2. **No background processing**: no BullMQ, no node-cron, no workers. The only async loop is a 3s
   `setInterval` poll in `realtime/websocket.routes.ts`. Detection rules, report generation, webhook
   delivery, cloud sync, alert evaluation, SLA breach, embeddings — all have no engine.
3. **RLS connection-pinning risk** (documented): `set_config(..., false)` on a 20-conn `postgres-js`
   pool can hand the follow-up Drizzle query a different connection → cross-tenant leak under load.
4. **Schema drift**: `packages/db/src/schema.ts` (49 tables) is behind `database/migrations/001_schema.sql`
   (richer columns: MFA, SSO, lockout on users; isolation/SLA fields on incidents; etc.). No drizzle-kit guard.
5. **Permission mismatches**: several write endpoints guard on `view:*` instead of `act:*`/`manage:*`
   (investigations POST/PATCH/notes, policies GET, reports POST, runbooks assign, detection-rules POST/PATCH/import).
6. **SaaS commercial layer absent**: no signup, billing, plans, metering, trials, licensing, tenant lifecycle.
7. **Integrations are CRUD-only stubs**: no connectors, no webhook delivery, no sync.
8. **AI is partly real, partly heuristic/static**: Copilot uses OpenAI when keyed (else canned fallback);
   RAG is keyword `ILIKE`, not vector (pgvector + `document_embeddings` are commented out); `/ai/*` endpoints
   return DB-derived heuristics with hardcoded confidence; `/ai/hunt-assist` is pure string templating.
